# Security policy

## Reporting a vulnerability

Please report suspected vulnerabilities through GitHub's
[private vulnerability reporting](https://github.com/sathwik-mamidi/open-order/security/advisories/new)
instead of a public issue. Include reproduction steps and the expected impact.

Security fixes are applied to the latest code on the default branch.

## Security boundaries

The public snapshot is a static, front-end-only product demonstration. It has
no checkout, payment processing, accounts, restaurant backend, or runtime API.
Those capabilities require a separate security design before this concept can
be used for real orders.
